# CStar Canon Glossary

Status: derived canon from CStar White Paper v0.3

This glossary keeps the world layer and system layer linked but not
interchangeable.

## Translation Table

| System term | World term | System meaning |
| --- | --- | --- |
| CStar Kernel | Star Core | Deterministic command, routing, registry, state, and authority layer. |
| Manifest / Skill Catalog | Grimoire / Spellbook | Catalog of capabilities, actions, constraints, and semantics. |
| Mobile Console | Sanctum | Primary operator interface and casting focus. |
| Spoke | Spell, familiar, relic, ward, tool, companion, or item | User-created module with bounded function and optional manifestation. |
| Skill | Castable ability | Specific callable function exposed by a spoke or core system. |
| Bead | Quest | Work item with intent, criteria, status, and validation path. |
| Engram | Rune / Lore / Memory | Recorded result, lesson, event, or validated history. |
| Warden | Guardian | Validator, scanner, policy gate, safety check, or quality check. |
| Loadout | Equipped build | Active configuration of spokes, slots, and manifestations. |
| Forge | Workshop / guild forge | Controlled self-building and community creation infrastructure. |
| Aura | Public presence | Privacy-preserving avatar, sigil, relic, and AR projection layer. |

## Term Discipline

- The world layer makes CStar legible, engaging, and personal.
- The system layer decides authority, execution, validation, and risk.
- A world term must never override a system term.
- A manifestation must never imply permissions not present in the manifest.
- Plain technical output must remain available for diagnostics, security, and
  operator review.

## Capability Terms

Capability tier describes action complexity, not operator level.

| Tier | Meaning |
| --- | --- |
| PRIME | Passive or read-only sensing. |
| SKILL | Bounded action. |
| WEAVE | Multi-step workflow. |
| SPELL | High-impact or high-authority action. |
| GREAT_WORK | Major composed operation with broad impact. |

Ritual is an execution pattern, not an authority class. Ascension is a rare
validated progression event, not a capability tier.
