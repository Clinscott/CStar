# Corvus Spatial AI Vision v0.1

Status: draft canon addendum to CStar White Paper v0.3.

Authority: This document clarifies the Corvus destination for spatial AI,
manifested spokes, and the Researcher frontier. It does not authorize
implementation, publication, live dispatch, public AR exposure, or authority
expansion by itself.

## Purpose

CStar already defines the doctrine: serious AI capability should feel magical
without hiding the mechanism. This addendum names the destination more plainly.

Corvus is building toward a world where real AI systems have spatial presence.
An operator writes, trains, equips, validates, and improves real software. That
software can then appear in the world as a compass, familiar, forge hammer,
shield, archive, relic, aura, tutor, sanctum, vessel, or other manifestation.

The manifestation is not the work. The manifestation is how the work becomes
legible to the operator and to trusted peers.

## Founder Vision

The end state is not a game pasted over an app. It is not a full MMO first. It
is not a cosmetic avatar layer that pretends to be meaningful.

The end state is a real operating framework where AI developers are legible as
builders of living systems. Their work has proof, memory, authority boundaries,
and visible form. Other operators can see selected public or trusted aspects of
those systems, understand what they are looking at, and interact with them
through consented contracts.

An operator may carry XO as a physical compass. In private AR, the compass can
open into a child-safe tutor, counsel surface, curriculum guide, and memory
interface. To a trusted peer, it may show only a privacy-safe companion icon,
guardian-approved learning seal, and chosen public identity. To the public, it
may reveal nothing more than a sigil or no projection at all.

That is the pattern.

The magic must be earned by real function.

## System Metaphor

The Corvus world language should map to concrete system surfaces:

| World term | System surface |
| --- | --- |
| Magus | Operator or builder. |
| Sanctum | Private command surface and personal operating space. |
| Castle | Repository, workspace, or personal spatial home for an operator's build. |
| Spoke | Bounded AI/software module with a manifest and authority contract. |
| Skill | Named callable capability. |
| Spell | User-facing invocation pattern for one or more skills. |
| Familiar | Spoke that observes, scouts, accompanies, or reports. |
| Relic | Narrow special-purpose capability with strong identity. |
| Warden | Validator, safety gate, policy guard, or adjudicator. |
| Engram | Recorded memory, decision, proof, or lesson. |
| Forge | Controlled build and review loop that turns accepted work into artifacts. |
| Aura | Privacy-filtered public or trusted spatial presence. |

The metaphor is allowed only when it improves clarity, emotional force, or
operator memory. It must not override manifests, policy, validation, or CStar
state.

## What The Spatial Layer Is

The spatial layer is a presentation and interaction layer for real CStar
objects:

- spokes with manifests
- skills with callable semantics
- beads with work state
- wardens with validation status
- engrams with recorded proof
- loadouts with active configuration
- auras with privacy-filtered presence
- sanctums with private operator context

The spatial layer may use mobile, desktop, browser, AR glasses, VR headsets,
physical activation objects, maps, 3D scenes, or simple 2D previews. The first
implementation should be boring enough to validate and evocative enough to
prove the direction.

## What The Spatial Layer Is Not

The spatial layer is not:

- authority
- trust
- validation
- secret state
- proof by appearance
- a replacement for CStar beads
- a replacement for Warden gates
- a reason to reactivate parked projects
- a reason to route work outside the Focus Charter

If a spoke looks like a blazing sword but only has observe authority, CStar
must make that distinction clear.

## Interoperability Principle

Users should be able to create almost anything. A spoke may look like a raven,
compass, dragonfly, cathedral, sword, forge, lantern, orbital satellite,
gaussian-splat memory room, mechanical talisman, or plain terminal panel.

Shared interaction still needs common ports:

- identity: what is this object?
- owner: who controls it?
- capability: what does it claim to do?
- authority: what is it allowed to affect?
- invocation: how can it be called?
- interaction: what contracts can it implement or consume?
- proof: what validates it?
- visibility: who may see which layer?
- privacy: what must never leak?

The Spoke Manifestation Protocol exists to preserve these ports without
flattening imagination.

## Reference Implementation: XO Compass

XO is the reference manifestation because its purpose is clear. It exists to
help the founder give his children better education and counsel. It is not a
generic game pet and not a toy.

Minimum useful XO Compass path:

1. Define XO's spoke manifest and child-safety authority.
2. Define an XO compass manifestation with public, trusted, and private views.
3. Create a simple local/web spatial preview of the compass opening into the
   tutor surface.
4. Bind invocation to a safe software action first.
5. Only later investigate physical activation objects such as a mechanical
   compass, NFC, Bluetooth, or device bridge.
6. Do not expose child data, curriculum state, identity, memory, or guardian
   controls through public aura.

XO proves the rule: the manifestation is powerful because the real purpose is
powerful.

## Personal Sanctums And Community Zones

CStar should not start by building one enormous shared world.

The first spatial homes should be personal sanctums: private operator spaces
where CStar can render a loadout, spokes, beads, engrams, wardens, and active
rituals. A sanctum can later map to a repository, profile, device, or physical
space.

Community zones should come later, after:

- public aura schema exists
- trusted peer visibility exists
- consent and privacy boundaries are enforced
- interaction contracts exist
- PvE trial alpha is stable
- Warden scoring is deterministic
- publication gates are clear

The public world is earned, not assumed.

## Geospatial And Immersive Direction

The long arc may include AR overlays, VR sanctums, map-scale presence, gaussian
splat scenes, world anchors, public guild spaces, and city-scale or global
layers. These are research directions, not immediate commitments.

Researcher should investigate them in service of the active charter:

- Does this help CStar become a better control plane?
- Does this make real spokes more understandable?
- Does this help XO, Moonshot, Forge, Skills, or Researcher itself?
- Does this preserve privacy and authority boundaries?
- Does this create a credible path from prototype to product?

The answer "this looks amazing" is not enough. The answer must connect to
purpose, proof, and focus.

## Near-Term Product Spine

The first practical spine should be:

1. Spoke Manifestation Protocol v0.1.
2. XO Compass manifestation spec.
3. Static/web spatial mock for XO Compass.
4. Generic Spoke Manifest Viewer that can render identity, authority, proof,
   visibility, and manifestation fields.
5. Corvus Forge manifestation mock showing Researcher, PMT, worker, finalizer,
   Warden, receipt, and PR proof as a spatial workflow.
6. Personal Sanctum sketch that shows active loadout, beads, wardens, engrams,
   and Focus Charter scope.

This sequence is enough to prove the direction without pretending the whole
world exists.

## Researcher Mandate

Researcher must become the scout for this frontier.

It should not merely collect interesting AR, VR, AI, game, or spatial computing
links. It should rank signal against Corvus purpose:

- CStar control-plane implications
- Researcher pipeline implications
- Forge and skill-loop implications
- XO Compass implications
- Moonshot simulation/visualization implications
- privacy and consent risks
- technical feasibility
- prototype value
- timing and platform readiness

Researcher should feed CStar with evidence-backed proposals, not ambient
curiosity.

## Stop Conditions

Stop or route to explicit operator decision if a proposed spatial feature:

- exposes private child, family, operational, security, device, or secret data
- implies authority from appearance
- makes parked projects active without Focus Charter reactivation
- requires public multiplayer before private sanctum or consent layers exist
- requires production deployment, device permissions, secrets, or live external
  account mutation
- turns Researcher into backlog creation instead of evidence intake
- asks Forge to build without CStar bead, issue, acceptance criteria, and
  validation path

## Guiding Line

CStar is the law. Corvus is the house. Researcher is the scout. Forge is the
smithy. Skills are the spells. Spokes are the works. Manifestation is how the
works become visible without lying about what they can do.
