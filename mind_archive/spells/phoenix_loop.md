# 🔱 SPELL: THE PHOENIX LOOP (v1.0)

## 💎 WHEN TO USE
Use when a technical restoration or forge cycle fails to satisfy the Triad of Verification. The Phoenix Loop ensures that the system does not abandon a sector until it is statistically stable.

## 🛠️ EXECUTION MODE
**Agent-Native**: Agents manually govern this recursive loop based on the feedback gates.

## 🧩 RECURSIVE PROTOCOL
1. **Initialize**: Execute `weave:restoration` on a target bead.
2. **Audit**: Calculate the Post-Forge Gungnir Score [Ω].
3. **Feedback Gate**: 
    - IF `Gungnir [Ω]` improved AND `Tests` are green: **RESOLVE**.
    - IF `Gungnir [Ω]` decreased OR `Tests` failed: **RECAST**.
4. **Recast**: Increase reasoning effort, adjust implementation, and re-execute.

## 🔄 WEAVE CHAIN
`weave:restoration` -> `skill:trace` -> `weave:evolve` (Loop)
