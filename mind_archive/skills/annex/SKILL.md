# 🔱 ANNEXATION PROTOCOL SKILL (v1.0)

## When to Use
- Use when auditing the repository for structural weaknesses, standards violations, and actionable annexation plans.


## MANDATE
Conduct comprehensive audits of the repository ("The Territory") to identify structural weaknesses, non-compliance with standards, and generate actionable battle plans.

## LOGIC PROTOCOL
1. **TERRITORY SCAN**: Execute Heimdall's Watch to crawl the realm for code and documentation.
2. **BREACH IDENTIFICATION**: Audit code for Linscott (Tests) and Torvalds (Quality) compliance.
3. **PLAN MATERIALIZATION**: Weave findings into a `ANNEXATION_PLAN.qmd` battle plan, preserving existing approvals.
4. **STRATEGIC LOGGING**: Report breach metrics (Critical vs. High) to the Sovereign HUD.

## USAGE
`cstar annex --scan [--path <dir>]`
`cstar annex --execute`
