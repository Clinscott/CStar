# 🔱 STERLING COMPLIANCE SKILL (v1.0)

## MANDATE
Audit the repository for compliance with the Sterling Mandate (Lore, Isolation, Audit).

## LOGIC PROTOCOL
1. **SECTOR IDENTIFICATION**: Map the file paths to be audited.
2. **MULTI-TIER AUDIT**: Execute the Sterling Auditor to evaluate Lore adherence, Unit Isolation, and Audit completeness.
3. **SOVEREIGNTY SCORE**: Calculate the compliance score and identify "Tarnished" sectors.
4. **REMEDIATION LOGGING**: Log identified gaps for the Mimir Campaign to resolve.

## CONSTRAINTS
- Every audited file must have a corresponding Gherkin contract and Unit Test.
- No "black box" logic allowed.

## USAGE
`cstar sterling --files <path1> <path2> ...`
